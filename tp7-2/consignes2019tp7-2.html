Introduction

Bootstrap, initialement développé par Twitter (et maintenant par un grand nombre de contributeurs externes), est une bibliothèque sous license libre, qui fournit un ensemble d'outils facilitant le travail des développeurs web :

  • Un ensemble de fichiers CSS qui apporte des styles par défaut à la plupart des éléments HTML, avec un aspect moderne.
  • Un système de grille dynamique, qui permet d'écrire très facilement la structure d'une page HTML "responsive".
  • Un ensemble de fichiers Javascript, qui permettent de dynamiser une page, par exemple en permettant la création d'onglets, d'alertes, etc.
  • Enfin, il existe de nombreux thèmes pour Bootstrap.

Dans ce TP, nous allons explorer quelques unes des possibilités offertes par Bootstrap.

Par où commencer ?

  • Créez un répertoire TP7-2 dans votre répertoire M1105
  • Dans ce répertoire TP7-2, créez un fichier essaibootstrap.html que vous initialiserez de cette manière (c'est un document Bootstrap minimal) :


<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width,initial-scale=1">
    <title>My awesome bootstrap page</title>
    <!-- Bootstrap -->
    <link href="bootstrap-3.3.5-dist/css/bootstrap.min.css" rel="stylesheet">
  </head>
  <body>
    <h1>
     Here, folks, shall live your code !
    </h1>

    <!-- jQuery (necessary for Bootstrap's JavaScript plugins) -->
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.11.3/jquery.min.js"></script>
    <!-- Bootstrap's JavaScript plugins -->
    <script src="bootstrap-3.3.5-dist/js/bootstrap.min.js"></script>
  </body>
</html>

Vous reconnaîtrez la structure de base d'un fichier HTML. Vous remarquerez que les styles CSS sont contenus dans le fchier bootstrap.min.css. Un ensemble de scripts Javascript sont également inclus: bootstrap.min.js, et il y a également une bibliothèque JavaScript très connue : jquery.min.js

Vou noterez également que parmi les métainformations, vous trouvez:
<meta name="viewport" content="width=device-width,initial-scale=1">
En effet, Bootstrap permet de créer facilement des designs adaptatifs (dits responsive en anglais - voir le TP précédent).

Téléchargez maintenant Bootstrap et dézippez-le dans votre dossier TP7-2.

Votre répertoire est maintenant prêt à accueilir des éléments Bootstrap !

Le modèle de grille "responsive"

Vous avez besoin de documentation; vous pouvez accéder à:

  • la documentation officielle de Bootstrap
  • le tutorial de W3Schools qui est également une bonne référence.

Grâce à un ensemble intelligent de classes CSS (attributs du type class="classe1 classe2 classe3"), il est très facile de construire une grille d'éléments dynamiques, qui se positionneront les uns à côté des autres, ou les uns au-dessus des autres, en fonction de la taille de la fenêtre.

Nous allons construire ici une page contenant quatre blocs de contenu. En fonction de la taille de la fenêtre on aimerait que les blocs soient agencés ainsi :


Ordinateur de bureau :

    | Bloc 1 | Bloc 2 | Bloc 3 | Bloc 4 |

Tablette :

    | Bloc 1  |  Bloc 2 |
    | Bloc 3  |  Bloc 4 |

Smartphone :

    |   Bloc 1   |
    |   Bloc 2   |
    |   Bloc 3   |
    |   Bloc 4   |

Pour ce faire, voici quelques indications :

  • Bootstrap utilise un système de grille qui se compose de plusieurs lignes pour le contenu (dans notre exmple, 1 seule ligne de 4 blocs)
  • Les lignes doivent être placées dans une division <div class="container-fluid"> ... </div> (full-width) ou dans une division <div class="container"> ... </div>(fixed-width)
  • Une ligne (<div class="row">... </div>)a une largeur de 12 colonnes.
  • une ligne se décompose en 1 ou plusieurs cellules (ou blocs) de contenu
  • Un bloc occupe un certain nombre de colonnes
  • On applique à un bloc des classes CSS bootstrap pour lui indiquer comment se positionner
  • Les noms de ces classes CSS bootstrap se forment sur le modèle suivant : col-ECRAN-LARGEUR, avec:
    • ECRAN: qui représente le type d'écran sur lequel vous voulez appliquer cette classe (xs pour smartphone, sm pour tablette, et md pour ordianteur de bureau)
    • LARGEUR: qui représente la largeur du bloc (largeur entre 1 et 12).
    • Par exemple, la classe class="col-md-3" signifie que le bloc doit avoir une largeur de 3/12 de la page sur un ordinateur de bureau. Vous appliquerez 3 classes à chacun des blocs.
    • Rmq: on peut avoir aussi une classe comme class="col-xs-12 col-sm-6 col-md-8"

Premier test:

Vous pouvez vous aider des explications du site Bootstrap Grid System et du site w3schools: Bootstrap Grids

Dans votre fichier essaibootstrap.html:

  • ajoutez une division <div class="container-fluid"> ... </div>
  • dans cette division, ajoutez une "ligne de grille" (<div class="row">... </div>) qui contiendra elle-même 4 blocs de contenu (<div class=" ">... </div>)
  • précisez les classes de ces 4 divisions et ajoutez-leur un contenu quelconque, par exemple un paragraphe "lipsum".

En changeant la largeur de la fenêtre de votre navigateur, vérifiez que les blocs s'agencent de la manière décrite ci-dessus.

Attention ! Si vos blocs n'ont pas tous la même hauteur, vous devrez ajouter des "retours à la ligne" (<div class="clearfix visible-sm-block"></div>), en remplaçant -sm- par la taille d'écran pour laquelle vous souhaitez insérer ce retour à la ligne. Plus d'explications sur le clearfix.

Menu de type "navbar"

Maintenant, nous allons créer un menu du type "navbar".

  • Copiez et collez le code suivant au début de votre division body :

    
 <nav class="navbar navbar-default">
 	<div class="container">
 		<div class="navbar-header">
 			<button type="button" class="navbar-toggle collapsed" data-toggle="collapse"
 			  data-target="#navbar" aria-expanded="false" aria-controls="navbar">
 				<span class="sr-only">Toggle navigation</span>
 				<span class="icon-bar"></span>
 				<span class="icon-bar"></span>
 				<span class="icon-bar"></span>
 			</button>
 			<a class="navbar-brand" href="#">Little One Apps</a>
 		</div>
 		<div id="navbar" class="collapse navbar-collapse">
 			<ul class="nav navbar-nav">
 			</ul>
 		</div>
 	</div>
 </nav>
  • Observez la structure du code rajouté, en particulier les identifiants (id) et les classes CSS utilisées. Un navbar créé avec Bootstrap doit avoir toujours à peu près la même structure.
  • Pour rajouter des liens au menu, créez des éléments li dans la liste ul. Chaque li devra contenir aussi un lien (<a>).
  • Sur Firefox, défilez vers le bas de la page. Observez que la barre de menu créée disparaît en haut. Nous pouvons changer ce comportement et "fixer" la barre. Rajoutez la classe navbar-fixed-top à la balise nav.
  • Sur Firefox, défilez vers le bas de la page à nouveau. Observez que maintenant la barre est toujours visible grâce à la classe navbar-fixed-top.
  • Avez-vous remarqué que cette dernière modification fait que le menu couvre le contenu de la page? Dans votre fichier TP7.css, ajoutez une marge intérieure haute de 50px à l'élément body. Cette configuration réglera le problème.
  • Redimensionnez maintenant votre fenêtre horizontalement. Que se passe-t-il ? Bootstrap 3 est un framework "mobile-fisrt", c'est-à-dire, il a été conçu en tenant compte d'abord les utilisateurs de dispositifs mobiles.

Voilà, notre menu est fait ! La prochaine étape concerne la mise en forme du contenu de la page.

Configuration de la grille

Nous avons presque fini. Voici un défi pour terminer : rajoutez de classes CSS de configuration de grille aux balises div qui sont dans la <div class="row">. Vous devez reproduire la configuration suivante.


Ordinateur de bureau :

    | Bloc 1 | Bloc 2 | Bloc 3 | Bloc 4 |

Tablette :

    | Bloc 1  |  Bloc 2 | Bloc 3 |
              |  Bloc 4 |

Smartphone :

    |   Bloc 1   |
    |   Bloc 2   |
    |   Bloc 3   |
    |   Bloc 4   |

Pour ce faire, voici quelques indications :

  • Vous aurez besoin d'utiliser l'ensemble de classes col-*-offset-* (par exemple, col-lg-offset-0). Voir la documentation pour plus d'information.
  • Observez que les balises <div class="container-fluid"> et <div class="row"> sont déjà créées. Pas besoin de les modifier !

Le design de votre page est terminé!

Pour aller plus loin : des composants Bootstrap

Nous allons maintenant voir d'autres composants pré-construits que Bootstrap propose.

Reprenez votre fichier essaibootstrap.html et dans vos blocs, ajoutez :

  • Un tableau (<table class="table table-hover">) qui contiendra quelques lignes (<tr>) et colonnes (<th> et <td>). Observez le style qui est appliqué au tableau
  • Des boutons (<button type="button" class="btn btn-default">Clic me</button>), en testant d'autres valeurs que btn-default (testez btn-primary, btn-success, btn-info, btn-warning, btn-danger, btn-link)
  • Des icônes dans le contenu de vos boutons (<span class="glyphicon glyphicon-search" aria-hidden="true"></span>). Liste des icônes
  • Des alertes (<div class="alert alert-success" role="alert">Super!</div>), en testant également alert-info, alert-warning et alert-danger

Un grand nombre de composants sont disponibles, voir la documentation.

Pour finir : un peu de dynamisme

Javascript est le langage de programmation qui permet d'animer les pages web : en les rendant interactives, en communiquant avec un serveur pour actualiser automatiquement le contenu (comme par exemple sur les webmails modernes), etc.

Bootstrap intègre une bibliothèqye Javascript qui permet de faire pas mal de "magie" automatiquement! Ajoutez par exemple sur votre page un modal (en plaçant son contenu à l'extérieur de votre container, et un bouton pour l'activer dans un de vos blocs). Un modal est une fenêtre qui apparait par-dessus la page pour donner une information à l'utilisateur. Pour cela, suivez la documentation qui fournit un exemple simple.

Vous pouvez encore aller plus loin avec Bootstrap: Bootstrap is the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web!
